# tokenizer阶段的coupling瓶颈

## 1、coupling tensor

coupling是我在设计本项目时的核心叙事，我们最理想的目标是找到一个能够对多任务泛化的优美的数学工具来描述EEG与fNIRS的配对关系。我对于task/source/phase/position aware机制并不信任，因为它意味着降低泛用性和新增数据集时的人工适配；如何构建一个天然对多任务成立的coupling机制？

> 在数学上我没有设计满足条件的配对机制的能力；添加z_context让模型自主总结上下文是一种可行的方案

## 2、codebook结构

虽然在下游任务上，理想情况应该直接使用token index的序列，就能够进行训练和推理。但是，在当前token之间共现关系不明确的情况下，对编码完成的token向量做进一步的检查，发现codebook中的向量有效秩较低。

tokenizer中continuous latent阶段还存在可以观测的跨模态信息，但是，在经过离散化之后，跨模态信息就被抹平了，如何让后续的tokenizer训练能够更好地保留跨模态信息？

## 3、EEG与fNIRS的tokenizer的互动

当前架构中EEG/fNIRS source/observation 四个编码器高度独立。在这种情况下，通过coupling loss就希望能够学到具有生理意义的token共现关系是否过于天真？有必要设计EEG与fNIRS的显式交互模块，比如交叉注意力吗？

## 4、tokenizer的最终要求

当前对tokenizer生成的token序列即出现稳定的具有生理意义的token共现关系的要求是合理的吗？如果我们希望tokenizer只是完成离散化步骤，在下游基于token index序列的预训练中进行token间生理相关性的学习，是否存在合理的训练方法？这一目标转移是合理的吗？

### 对tokenizer阶段配对和下游任务阶段配对的对比分析

- 在tokenizer训练阶段tokenizer能够接收的数据是原始的EEG与fNIRS信号，信息丰富但是信噪比低；当前tokenizer的训练中，没有显式的设计EEG与fNIRS tokenizer之间的信息互动，而是希望通过coupling loss的设计，约束各个tokenizer来生成有生理意义的EEG与fNIRS token共现关系。

- 但是，在过去的不断修改尝试中，coupling loss的设计始终没有带来理想的效果。我们尝试了从较弱的生理先验到更强的约束，并在最新的实验中测试可以区分上下文的coupling loss设计。我现阶段的主要疑问是，coupling loss的约束在数学上是否偏弱？它是不是不具备指导tokenizer发现生理耦合关系应该具有的能力？

- 如果需要在下游任务阶段进行生理耦合发现，只需要处理EEG与fNIRS已经编码完成的token index序列，理论上具有更高的信噪比；但是，在此前的测试实验中，从tokenizer中的continuous latent到离散化的token index，跨模态的信息严重丢失；只接收token index的下游任务预训练过程，应该以什么方式来提供可解释的生理耦合信息？而且，EEG与fNIRS token序列已经固定，会导致某个EEG token之后的数秒，高频出现某个fNIRS token这种，最接近两个生理模式天然对应的叙事直接失效。

## 5、coupling loss存在的问题

### coupling loss 数学上偏弱

梯度路径过长。coupling loss的梯度需要经过以下路径才能影响编码器参数：

coupling_logits[lag, e, f]
  → softmax → P(f|e, lag)
  → einsum with soft EEG/fNIRS assignment probs
  → straight-through estimator (hard argmax → one-hot)
  → assignment_logits
  → encoder parameters

这条路径的核心问题是 straight-through estimator：VQ 的 argmax 操作不可微，STE 用一个恒等映射近似梯度。这意味着：

- Token 分配决策（"这个 patch 被分配给 token 7 还是 token 23"）不接收来自 coupling loss 的直接梯度
- Coupling loss 只能通过软分配概率的连续松弛来间接影响，而这种松弛与硬分配之间的 mismatch 随训练加深

**信息论解释**：当 source codebook 为 K=64 时，每个 token index 只携带 log₂(64) = 6 bits 的信息。而 continuous latent 是 48-128 维的连续向量。VQ 步骤是极端的信息瓶颈——它将高维连续表示压缩到极其有限的信息通道中。Coupling loss 要求的"EEG token A 出现后 4-6 秒应出现 fNIRS token B"是一种精细的条件依赖结构，6 bits/token 的信道容量根本不足以编码这种结构，除非 encoder 在训练中被迫将耦合相关的信息优先分配给这些 bits。

当前的 encoder 没有这种压力机制。 EEG encoder 和 fNIRS encoder 是完全独立的——它们各自优化 reconstruction + VQ commitment + codebook balance，只有在最终的 coupling loss 中才发生交互。这意味着：

1. Encoder 学到的 token 分组主要服务于重构质量，而非跨模态可预测性
2. Coupling loss 是一个"事后"约束——它只能重新排列已经形成的 token→quantized vector 的映射（通过 coupling_logits），但无法显著改变 encoder 如何将原始信号映射到 token
3. 梯度太弱、太间接，不足以重塑离散化决策

> 一个有用的思维实验：如果你的 EEG encoder 和 fNIRS encoder 产生的是完全随机但重构良好的 token，coupling loss 能通过 STE 梯度将它们重新组织成有意义的耦合结构吗？答案是不能——因为 STE 梯度不能提供足够的信号来改变"哪个 patch 被分配给哪个 token"的决策，它只能微调 coupling_logits 本身。

### VQ离散化过程没有跨模态信息保存机制

从 information ladder 数据可以看出，continuous latent 中 cross-modal MI 从 raw signal 的 ~0.03-0.06 提升到了 ~0.15-0.17——这说明 encoder 确实在学习提取跨模态相关的特征。但离散化后 MI 骤降至 ~0.03-0.05。

这意味着 VQ codebook 的学习过程（EMA 更新 + commitment loss）天然倾向于保留重构相关信息而丢弃跨模态信息，因为：

- VQ 的 codebook 更新完全由重构梯度驱动（通过 commitment loss）
- Codebook balance loss 推动均匀使用——这与"让某些 token 专注于耦合相关信息"形成对冲
- 没有任何机制告诉 VQ："请在量化时保留 EEG latent 中与 fNIRS 相关的方向"

### 独立编码器架构是结构性问题

当前架构（factorized_labram_vqnsp.py:1481-1489）中：

eeg_encoded = self._encode_modality(eeg, ...)  # 完全独立
fnirs_encoded = self._encode_modality(fnirs, ...)  # 完全独立
eeg_source = self.eeg_source_proj(eeg_encoded)
fnirs_source = self.fnirs_source_proj(fnirs_encoded)

EEG encoder 和 fNIRS encoder 之间没有任何信息交换。它们各自独立地将原始信号编码为 latent，然后独立地投影到 codebook 空间。Coupling 只发生在离散化之后。

这种设计下，coupling loss 必须完成一个几乎不可能的任务：仅通过调整 coupling_logits 和极弱的 STE 梯度，让两个从未"见过"对方的 encoder 产生对齐的离散表示。
