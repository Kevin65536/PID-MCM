## 3. 各阶段分解

### 阶段 1：范围分解（每轮 1 个 agent）

范围分解 agent 将研究问题分解为 5 个互补的搜索角度，每个角度带有特定的搜索查询和理由。

**第一轮范围分解**: 覆盖范式 A（tokenizer耦合）和范式 B（下游发现）：
1. 多模态 VQ-VAE 耦合梯度
2. 跨模态 tokenizer 交互架构
3. 从离散事件序列中发现时序模式
4. VQ 瓶颈下的信息保留
5. 来自语音/视频/生物医学多模态 tokenization 的经验教训

**第二轮范围分解**: 专门聚焦于批判性地评估 STE 梯度间隙声明：
1. 独立的 STE 梯度间隙分析（不引用 Huh et al.）
2. EMA/cosine 量化器梯度动态
3. 通过 VQ 的信息瓶颈
4. 跨模态 VQ-VAE 辅助损失的成功案例
5. 神经科学文献中的 EEG-fNIRS 互信息

### 阶段 2：搜索（每轮 5 个 agent 并行）

每个搜索 agent 使用 WebSearch + WebFetch 为其角度查找相关论文。每个 agent 产出结构化结果，包含：
- URL、标题、摘要片段以及每篇找到论文的相关性评级
- 每个角度约 5 篇论文，附有详细注释

### 阶段 3：获取（21-23 个 agent 并行）

专用的获取 agent 检索已发现论文的全文 PDF。`tool-results/` 中的 53 个 PDF 是这些获取操作的缓存输出。每个获取 agent 处理 1-3 篇论文。

### 阶段 4：验证（v0→v1→v2，约 75 个 agent 总计）

**这是"投票"机制。** 从来源中提取的声明经过三轮验证：

- **v0（初始提取）**: 一个 agent 从每篇论文中提取 3-5 条具体的、带引用的声明，并标注重要性评级（核心/支撑/边缘）
- **v1（对抗性质疑）**: 另一个不同的 agent 尝试通过寻找反证或逻辑缺陷来反驳每条声明
- **v2（裁决）**: 第三个 agent 评估 声明 vs 反驳 并做出最终裁决

标签命名规范: `v{N}:{声明文本截断至50字符}`

### 阶段 5：综合（每轮 1 个 agent）

综合 agent 将所有已确认的声明整合为连贯的叙述，包含：
- 最终研究发现（4-6 项）
- 被驳回的声明摘要
- 未来调查的开放性问题
- 方法论上的注意事项
